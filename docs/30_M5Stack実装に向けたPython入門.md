# M5Stack実装に向けたPython入門

このページは、**プログラミング入門I**で学んだ内容に、あと少しだけ「M5Stack のサンプルを
読んだり書いたりするために必要な考え方」を足すための補足教材です。

プログラミング入門I では、変数・計算・`if`・`while`・`print`・関数の作り方（`def`）・
リストなどを学びました。M5Stack のサンプルは、その知識に加えて、**どのサンプルにも
共通で出てくる3つの新しい書き方**を使っています。ここではその3つだけをやさしく説明します。

1. **import** … 道具（モジュール）を取り込む
2. **関数・メソッドの呼び出し** … 取り込んだ道具の機能を「.（ドット）」で使う
3. **try / except** … プログラムを安全に止める

以降のコードは、実際のサンプル（`Level1_samples/` と `m5stack_samples/`）から引用しています。

---

## 0. プログラムの全体の流れ（`setup()` と `loop()`）

まず、**すべてのサンプルに共通する骨組み**から説明します。どのサンプルも、
`setup()` と `loop()` という2つの関数を作り、次の形で動きます。

```python
def setup():
    # 起動時に「一度だけ」実行する準備（ピンの用意など）
    ...

def loop():
    # 「繰り返し」実行する処理（センサー読み取り・LED点灯など）
    ...

setup()                     # ① 最初に setup() を1回だけ呼ぶ
try:
    while True:             # ② そのあと loop() をずっと繰り返す
        loop()
except KeyboardInterrupt:   # ③ Ctrl-C で止めたら、後始末をして終了
    print("終了しました")
```

`setup()` と `loop()` は、みなさんが入門I で学んだ「自分で作る関数（`def`）」そのものです。
特別な命令ではなく、**関数の名前を setup・loop という決まった名前にそろえているだけ**です。

### なぜ setup / loop に分けるのか（Arduino に合わせています）

M5Stack のようなマイコンは、**Arduino** という開発環境でよくプログラムされます。
Arduino では、プログラムを必ず

- `setup()` … 電源を入れたときに一度だけ動く（初期設定）
- `loop()` … そのあと電源が切れるまで、ずっと繰り返し動く

という2つに分けて書く決まりになっています。
このサンプル集は、**その Arduino の書き方に合わせて** `setup()` と `loop()` に分けています。
こうしておくと、「最初に一度だけやること」と「ずっと繰り返すこと」がはっきり分かれて読みやすく、
Arduino で書かれた M5Stack の作例とも見比べやすくなります。

### 全体の流れ（まとめ）

1. **`setup()` を1回だけ呼ぶ** … センサーやLEDを使う準備をする。
2. **`while True:` の中で `loop()` をずっと繰り返す** … 本体の仕事（測る・光らせる・表示する）を続ける。
3. **`Ctrl-C` を押すと止まる** … このとき `try / except` が働いて、きれいに終了する（詳しくは下の「3. try / except」）。

この骨組みが分かれば、あとは `setup()` と `loop()` の中身を読むだけです。
その中身を読むために必要なのが、次の3つ（import／メソッド呼び出し／try・except）です。

## 1. import ── 道具箱を取り込む

Python には、最初から使える命令（`print` など）のほかに、**モジュール**という「道具箱」が
たくさん用意されています。M5Stack のセンサーやLED、時間待ちなどは、道具箱の中に入っていて、
使う前に **`import`（インポート）** で取り込む必要があります。

たとえば `magnetic_switch.py`（磁気スイッチ）の先頭は、こうなっています。

```python
from machine import Pin   # machine という道具箱から、Pin という道具を取り込む
import time               # time という道具箱まるごとを取り込む
```

import には2つの書き方があります。

`import time` のように**道具箱ごと**取り込むと、使うときは「道具箱名.機能」と書きます。

```python
import time
time.sleep(0.2)    # time 道具箱の中の sleep（0.2秒待つ）を使う
```

`from machine import Pin` のように**道具箱から必要な道具だけ**取り込むと、
その名前（`Pin`）をそのまま書けます。

```python
from machine import Pin
p = Pin(9, Pin.IN)   # 道具箱名を付けずに Pin を使える
```

> プログラミング入門I の「turtle モジュール」で `import turtle` と書いたのと同じ考え方です。
> `print` のような命令と違って、モジュールの機能は **import しないと使えません**。

## 2. 関数・メソッドの呼び出し ── ドットで機能を使う

みなさんは `def` で自分の関数を作り、それを呼び出す方法を学びました。
M5Stack のサンプルでは、それに加えて、**モジュールや「モノ（オブジェクト）」が最初から
持っている関数**を呼び出します。「モノに付いている関数」を **メソッド** と呼び、
`モノ.メソッド()` のように **ドット（.）** で呼び出します。

`light_sensor.py`（明るさセンサー）を見てみましょう。

```python
from machine import Pin, ADC

light = ADC(Pin(9))          # ADC という「モノ」を1つ作って、変数 light に入れる
light.atten(ADC.ATTN_11DB)   # light というモノに「測る範囲を設定して」とお願いする
value = light.read_u16()     # light に「今の値を教えて」と聞く → 数値が返ってくる
```

ここでのポイントは2つです。

まず `ADC(Pin(9))` のように**モノを作って変数に入れる**こと。次に、その変数に対して
`light.read_u16()` のように**ドットでメソッドを呼ぶ**ことです。`read_u16()` は
「関数は値を返す」で学んだとおり、呼ぶと明るさの数値（0〜65535）を**返して**くれます。

かっこの中に書く値（`Pin(9)` の `9` や `sleep(0.2)` の `0.2`）は、
関数の「実引数」で学んだものと同じで、**関数に渡す情報**です。

CoreS3 本体の機能も同じ書き方です（`speaker_tone.py`）。

```python
from M5 import *

Speaker.begin()          # スピーカーを使い始める
Speaker.setVolume(128)   # 音量を設定する（0〜255）
Speaker.tone(440, 300)   # 440Hz の音を 300ミリ秒 鳴らす
```

`Speaker` という道具に対して、`.begin()` `.setVolume(...)` `.tone(...)` と
**ドットで次々にお願いしている**だけです。難しく見えても「モノにドットで頼む」形に慣れれば、
どのサンプルも同じパターンで読めます。

## 3. try / except ── プログラムを安全に止める

M5Stack のサンプルは、センサーを読み続けたり LED を光らせ続けたりするため、
**`while True:` でずっと繰り返し**ます。止めるときは、PC のターミナルで **Ctrl-C** を押します。

このとき Python の内部では `KeyboardInterrupt`（キーボードで中断された、という**例外**）が
発生します。**例外**とは「そのままだとエラーで止まってしまう出来事」のことです。
それを **`try` / `except`** で受け止めると、エラーで乱暴に止まる代わりに、
自分で決めた後始末をしてから、きれいに終われます。

どのサンプルにも共通で、末尾はこの形になっています。

```python
setup()                       # 最初に一度だけ準備をする
try:
    while True:               # ずっと繰り返す
        loop()                # センサー読み取りやLED点灯など
except KeyboardInterrupt:     # Ctrl-C が押されたら…
    print("終了しました")      # メッセージを出して、きれいに終わる
```

`try:` の中身を実行していて、途中で例外（ここでは Ctrl-C）が起きると、
すぐに `except:` の中へ飛びます。これで「Ctrl-C で止めても、ちゃんと『終了しました』と
表示して終わる」動きになります。

例外は Ctrl-C 以外にも使えます。`dht11.py`（温湿度センサー）では、読み取りに失敗したときの
`OSError` を受け止めて、プログラムが止まらないようにしています。

```python
try:
    sensor.measure()                       # 温度・湿度を測る（たまに失敗する）
    print(sensor.temperature(), sensor.humidity())
except OSError:                            # 失敗したら…
    print("読み取り失敗（配線・電源を確認）")   # 知らせて、次の回に進む
```

> まとめると、`try/except` は「失敗しそうな処理を try に入れておき、失敗したら except で
> 受け止める」という安全網です。M5Stack のように**ずっと動き続けるプログラム**では、
> 止め方（Ctrl-C）とセットでとても役に立ちます。
